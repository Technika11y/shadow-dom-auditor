# Security Policy

## Scope & intended use

Shadow-DOM Auditor is a **defensive** auditing tool. It loads pages and inspects their DOM; it
performs no exploitation. Only run it against properties you own or are explicitly authorized to
test.

## Reporting a vulnerability

If you find a security issue **in this tool**, please report it privately rather than opening a
public issue:

- Use GitHub's **Report a vulnerability** (Security → Advisories) on this repository, or
- email the maintainer listed in the repo profile.

Please include a description, reproduction steps, and impact. Target response: **5 business days**
to acknowledge, and a coordinated disclosure timeline agreed with you before any public write-up.

This mirrors the responsible-disclosure posture the Technika11y suite advocates
(see the Incident Disclosure Toolkit).
